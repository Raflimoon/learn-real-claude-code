# 上下文即资源：三级压缩续命术

> 七种独立策略按优先级顺序触发：snip → microcompact → collapse → summarize。

## 概述

Context window 是 agent 最珍贵也最有限的资源。一个长时间运行的 coding session 很容易在几十次工具调用后就接近限制——每次文件读取、每次 bash 输出都在消耗这个有限的空间。如果不加管理，session 就会崩溃，丢失所有工作上下文。

Claude Code 构建了一套精细的七策略压缩系统，在 context 超过 95% 时自动触发。关键在于这七种策略按优先级顺序执行——每种策略都比上一种更激进，只有在温和的策略不够用时才升级到更激进的手段。这最大限度地保留了对话的连续性和信息密度。

## 触发条件

```typescript
// tokenBudget.ts
const COMPACTION_THRESHOLD = 0.95; // 95% 触发

function getCompactionNeed(messages: Message[]): CompactionLevel {
  const usage = estimateTokenUsage(messages);
  const ratio = usage / getCurrentContextLimit();

  if (ratio < COMPACTION_THRESHOLD) return 'none';
  if (ratio < 0.97) return 'mild';
  if (ratio < 0.99) return 'moderate';
  return 'aggressive';
}
```

压缩级别决定了执行哪些策略。

## 七种压缩策略

```
优先级  策略          目标                        激进程度
──────  ────────────  ──────────────────────────  ────────
  1     snip          删除重复的系统消息            最低
  2     microcompact  总结最近的工具结果            低
  3     collapse      折叠长文件读取内容            中
  4     deduplicate   去除重复的工具调用            中
  5     truncate      截断过长的单条消息            中高
  6     summarize     整个对话摘要                  高
  7     reset         保留摘要，丢弃全部历史        最高
```

### 策略 1：snip — 删除重复系统消息

最温和的策略。Anthropic API 允许在消息历史中插入系统消息（`role: "system"` 在中间位置），但重复的系统消息会浪费大量 token：

```typescript
function snip(messages: Message[]): Message[] {
  const seen = new Set<string>();

  return messages.filter(msg => {
    if (msg.role !== 'system') return true;
    if (seen.has(msg.content)) return false; // 删除重复
    seen.add(msg.content);
    return true;
  });
}
```

### 策略 2：microcompact — 总结最近工具结果

工具结果往往是最大的 token 消耗来源。`microcompact` 把最近的工具结果替换为 AI 生成的摘要：

```typescript
async function microcompact(messages: Message[]): Promise<Message[]> {
  // 找到最近的工具结果
  const recentToolResults = getRecentToolResults(messages, { count: 5 });

  for (const result of recentToolResults) {
    if (tokenCount(result.content) > 500) {
      // 用一个小的 AI 调用生成摘要
      result.content = await summarizeToolResult(result);
    }
  }

  return messages;
}
```

### 策略 3：collapse — 折叠文件读取

文件读取的完整内容通常不需要在整个会话中保留：

```typescript
function collapse(messages: Message[]): Message[] {
  return messages.map(msg => {
    if (isFileReadResult(msg) && isOlderThan(msg, { turns: 3 })) {
      // 把完整文件内容替换为折叠摘要
      return {
        ...msg,
        content: `[文件内容已折叠：${extractFilePath(msg)}，${lineCount(msg.content)} 行]`,
      };
    }
    return msg;
  });
}
```

注意 `isOlderThan({ turns: 3 })` 条件——最近 3 轮的文件读取不会被折叠，因为它们很可能仍然相关。

### 策略 6：summarize — 整个对话摘要

当轻量策略不够用时，`summarize` 会把整个对话历史压缩为一条摘要消息：

```typescript
async function summarize(messages: Message[]): Promise<Message[]> {
  // 保留最近 N 条消息（最新的内容最重要）
  const recentMessages = messages.slice(-10);
  const olderMessages = messages.slice(0, -10);

  if (olderMessages.length === 0) return messages;

  const summary = await callClaude({
    system: '你是一个对话摘要助手，将以下对话历史压缩为简洁摘要，保留所有重要决策、发现和上下文',
    messages: olderMessages,
  });

  return [
    { role: 'user', content: '[对话历史摘要]' },
    { role: 'assistant', content: summary },
    ...recentMessages,
  ];
}
```

关键设计：**保留最近的消息，先压缩旧的内容。** 用户最近做的事情比几十轮前的内容更重要。

## 策略执行顺序

```typescript
async function autoCompact(
  messages: Message[],
  level: CompactionLevel
): Promise<Message[]> {
  let result = messages;

  // 轻度压缩：只运行前三种策略
  if (level === 'mild') {
    result = snip(result);
    result = await microcompact(result);
    result = collapse(result);
    return result;
  }

  // 中度压缩
  if (level === 'moderate') {
    result = snip(result);
    result = await microcompact(result);
    result = collapse(result);
    result = deduplicate(result);
    result = truncate(result);
    return result;
  }

  // 激进压缩：运行全部策略
  result = snip(result);
  result = await microcompact(result);
  result = collapse(result);
  result = deduplicate(result);
  result = truncate(result);
  result = await summarize(result); // 最激进
  return result;
}
```

## 保留原则

所有策略都遵循同一个核心原则：**优先保留近期内容，优先丢弃旧的工具结果。** 近期内容代表当前工作的直接上下文；工具结果（尤其是文件读取）通常可以重新获取，而对话中的推理过程和决策则无法重新生成。

## 关键收获

- 七种压缩策略按优先级顺序触发，温和手段优先执行，只有当释放的空间不足时才升级到更激进的策略。
- `snip` 删除重复系统消息；`microcompact` 用 AI 摘要最近工具结果；`collapse` 折叠超过 3 轮之前的旧文件读取内容。
- `summarize` 是最激进的可逆策略，将旧对话替换为 AI 生成的摘要，同时保留最近 N 条消息以维持当前工作上下文。
- 所有策略都遵循同一核心原则：优先保留近期内容，优先压缩旧的工具结果——因为工具结果可以重新获取，而推理过程和决策无法重新生成。
- 压缩在 context 使用率超过 95% 时自动触发，嵌入在 agent 循环内部，对上层调用方完全透明。
