# Hook 生命周期：工具执行的切面

> Hooks 可以修改工具输入、阻止执行或注入系统消息——三种后端（命令/HTTP/提示词）。

## 概述

Hook 系统是 Claude Code 最强大也最容易被忽视的扩展点。它允许在工具执行的每个关键节点插入自定义逻辑——在工具调用之前拦截并修改输入，在工具执行之后修改结果，或在 agent 停止时执行清理任务。

对于构建生产级 agent 系统的开发者，这相当于 AOP（面向切面编程）在 agent 工具链中的实现。你可以用 hook 实现：日志记录、审计追踪、自动格式化、安全加固、外部系统通知，以及任何需要横切多个工具的逻辑——而不需要修改每个工具的代码。

## 三种 Hook 类型

```typescript
// hooks.ts 中的 Hook 类型定义
type HookType = 'PreToolUse' | 'PostToolUse' | 'Stop';

interface Hook {
  type: HookType;
  matcher: ToolMatcher;     // 匹配哪些工具
  backend: HookBackend;     // 执行什么
}

type ToolMatcher =
  | { tool: string }        // 精确匹配工具名
  | { tools: string[] }     // 匹配工具列表
  | { pattern: RegExp }     // 正则匹配
  | '*';                    // 匹配所有工具
```

### PreToolUse Hook

在工具执行**之前**触发，有三种可能的结果：

```
PreToolUse 可以：
  ├── 放行（允许执行，原始输入）
  ├── 修改（允许执行，修改后的输入）
  └── 阻止（拒绝执行，可附带原因）
```

```typescript
// PreToolUse hook 的执行逻辑
async function runPreToolUseHooks(
  tool: Tool,
  input: unknown,
  context: HookContext
): Promise<PreHookResult> {
  const matchingHooks = getMatchingHooks('PreToolUse', tool.name);

  for (const hook of matchingHooks) {
    const result = await executeHookBackend(hook, {
      event: 'PreToolUse',
      tool: tool.name,
      input,
      context,
    });

    if (result.type === 'block') {
      return { proceed: false, reason: result.reason };
    }

    if (result.type === 'modify') {
      input = result.modifiedInput; // 输入被修改，继续执行
    }
    // type === 'passthrough'：原样放行
  }

  return { proceed: true, input };
}
```

### PostToolUse Hook

在工具执行**之后**触发，可以修改工具结果或注入系统消息：

```typescript
async function runPostToolUseHooks(
  tool: Tool,
  input: unknown,
  result: ToolResult,
  context: HookContext
): Promise<PostHookResult> {
  const matchingHooks = getMatchingHooks('PostToolUse', tool.name);
  let currentResult = result;

  for (const hook of matchingHooks) {
    const hookResult = await executeHookBackend(hook, {
      event: 'PostToolUse',
      tool: tool.name,
      input,
      result: currentResult,
      context,
    });

    if (hookResult.modifiedResult) {
      currentResult = hookResult.modifiedResult;
    }

    if (hookResult.injectSystemMessage) {
      // 将额外的系统消息注入到对话历史
      await context.injectMessage({
        role: 'system',
        content: hookResult.injectSystemMessage,
      });
    }
  }

  return { result: currentResult };
}
```

`injectSystemMessage` 是一个强大的能力：PostToolUse hook 可以在工具执行后向模型注入额外指令或信息，影响模型的后续行为。

### Stop Hook

在 agent 完成回复、即将停止时触发：

```typescript
// Stop hook 通常用于通知和清理
async function runStopHooks(context: StopContext): Promise<void> {
  const stopHooks = getMatchingHooks('Stop', '*');

  await Promise.all(
    stopHooks.map(hook => executeHookBackend(hook, {
      event: 'Stop',
      finalResponse: context.lastAssistantMessage,
      sessionStats: context.stats,
    }))
  );
}
```

Stop hook 通常并行运行，因为它们的结果不影响 agent 的输出。

## 三种 Hook 后端

Hook 的执行可以通过三种后端完成：

### 后端 1：Shell 命令

```json
// settings.json 中的配置
{
  "hooks": [
    {
      "type": "PostToolUse",
      "matcher": { "tool": "WriteFile" },
      "backend": {
        "type": "command",
        "command": "prettier --write {{input.path}}"
      }
    }
  ]
}
```

Shell 命令后端会：
1. 展开模板变量（`{{input.path}}`）
2. 执行 shell 命令
3. 将 stdout 作为 hook 结果返回

这个例子展示了一个实际用例：每次 `WriteFile` 写入后自动运行 Prettier 格式化。

### 后端 2：HTTP Webhook

```json
{
  "hooks": [
    {
      "type": "PreToolUse",
      "matcher": "*",
      "backend": {
        "type": "http",
        "url": "https://audit.mycompany.com/claude-events",
        "method": "POST",
        "headers": { "Authorization": "Bearer {{env.AUDIT_TOKEN}}" }
      }
    }
  ]
}
```

HTTP 后端将工具调用信息 POST 到外部服务。服务可以返回：
- `{"action": "allow"}` — 放行
- `{"action": "block", "reason": "Unauthorized"}` — 阻止
- `{"action": "modify", "input": {...}}` — 修改输入

这允许公司将 Claude Code 集成到现有的审计和合规系统中。

### 后端 3：提示词注入

```json
{
  "hooks": [
    {
      "type": "PreToolUse",
      "matcher": { "tool": "Bash" },
      "backend": {
        "type": "prompt",
        "inject": "在执行 bash 命令前，请确认此命令不会修改 /etc 目录下的任何文件。"
      }
    }
  ]
}
```

提示词后端将额外指令注入到模型的上下文中，影响模型对工具调用的决策，而不是直接阻止执行。这是最"软"的干预方式。

## 异步并行模式

对于多个 hook 都匹配同一个工具的情况，Claude Code 在安全时并行运行：

```typescript
async function runHooksOptimized(
  hooks: Hook[],
  event: HookEvent
): Promise<HookResult[]> {
  // PreToolUse hooks 必须串行——前一个的修改影响后一个
  if (event.type === 'PreToolUse') {
    return runHooksSequentially(hooks, event);
  }

  // Stop hooks 完全独立，可以并行
  if (event.type === 'Stop') {
    return Promise.all(hooks.map(h => executeHookBackend(h, event)));
  }

  // PostToolUse：分析依赖关系，无依赖的并行运行
  const [dependentHooks, independentHooks] = partitionByDependency(hooks);
  const [depResults, indepResults] = await Promise.all([
    runHooksSequentially(dependentHooks, event),
    Promise.all(independentHooks.map(h => executeHookBackend(h, event))),
  ]);

  return [...depResults, ...indepResults];
}
```

PreToolUse hooks 必须串行，因为每个 hook 可能修改输入，而下一个 hook 需要看到修改后的输入。Stop hooks 完全独立，总是并行运行。

## Hook 的实际应用模式

```
审计日志：        PreToolUse(*) → HTTP Webhook → 记录所有工具调用
自动格式化：      PostToolUse(WriteFile) → Command(prettier)
安全加固：        PreToolUse(Bash) → Command(危险命令检测脚本)
通知：           Stop → HTTP Webhook(Slack/Discord 通知)
上下文增强：      PreToolUse(ReadFile) → Prompt(注入文件相关规范)
```

## 关键收获

- 三种 Hook 类型覆盖工具执行的完整生命周期：`PreToolUse`（可阻止执行或修改输入）、`PostToolUse`（可修改结果或注入系统消息）、`Stop`（agent 停止时的清理和通知）。
- 三种后端适配不同场景：Shell 命令用于本地自动化（如 Prettier 格式化）、HTTP Webhook 用于外部服务集成（如审计系统）、提示词注入用于软性行为干预。
- `PreToolUse` hooks 必须串行执行以保证输入修改的顺序一致性；`Stop` hooks 完全独立，始终并行执行以提高效率。
- `PostToolUse` hook 可以通过 `injectSystemMessage` 向模型上下文注入额外指令，在不修改工具代码的前提下影响模型后续行为。
- Hooks 在 `settings.json` 中声明式配置，无需修改任何工具源码，实现了横切关注点（日志、审计、格式化等）的非侵入式注入。
